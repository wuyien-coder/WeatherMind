# WeatherMind - AI 天气助手智能体

## 📋 项目概述

基于 **LangChain + LangGraph** 构建的智能天气助手，展示了 Agent 开发的核心技术和产品设计思维。

**技术栈**：Python 3.10+ | LangChain | LangGraph | Qwen3.5-Plus | Open-Meteo API

---

## 🎯 核心

- ✅ **用户场景设计**：智能识别实时天气/预报/对比等需求
- ✅ **容错体验**：定位失败时友好引导手动输入
- ✅ **场景化建议**：穿衣、出行、运动等个性化推荐
- ✅ **极端天气预警**：主动提醒危险天气
- ✅ **交互式帮助**：内置 use case 引导

## 🏗️ 技术架构

### Agent 核心组件
```
┌─────────────────────────────────────┐
│         LLM (qwen3.5-plus)          │
│      推理引擎 + Function Call       │
└──────────────┬──────────────────────┘
               │
    ┌──────────┴──────────┐
    │                     │
┌───▼────────┐   ┌────────▼──────┐
│ 定位工具   │   │  天气工具     │
│ • ipapi.co │   │ • Open-Meteo  │
│ • ip-api   │   │ • 实时 +7 天   │
│ • ipinfo   │   │ • 降水概率    │
└────────────┘   └───────────────┘
```

### 关键代码模块

#### 1. 工具定义（@tool 装饰器）
- `get_user_location()`: 多源 IP 定位
- `get_weather_for_location()`: 全球天气查询

#### 2. Agent 编排（create_agent）
```python
agent = create_agent(
    model=model,              # LLM 配置
    system_prompt=SYSTEM_PROMPT,  # 系统提示工程
    tools=[...],              # 工具链
    context_schema=Context,   # 上下文管理
    checkpointer=checkpointer # 会话记忆
)
```

#### 3. 上下文管理（Context）
```python
@dataclass
class Context:
    user_id: str                    # 用户标识
    preferences: Dict[str, Any]     # 用户偏好（扩展性）
```

## 💡 产品设计细节

### 1. 系统提示工程
- **角色定位**：专业天气助手
- **能力边界**：明确说明可用工具和限制
- **交互规范**：先完整数据 → 再精炼总结

### 2. 用户体验优化
- **帮助系统**：输入 `help` 查看示例
- **优雅降级**：定位失败 → 引导手动输入城市
- **错误反馈**：详细日志 + 友好提示

### 3. 可扩展性设计
- **多用户支持**：thread_id 隔离会话
- **偏好配置**：Context.preferences 预留扩展
- **API 抽象**：易于替换天气数据源
